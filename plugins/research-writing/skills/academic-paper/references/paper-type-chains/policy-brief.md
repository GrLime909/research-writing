# Paper Type: Policy Brief

Corresponds to `paper_structure_patterns.md` Pattern 5.

## Argument Chain

`problem -> evidence -> options analysis -> recommendation -> implementation -> metrics`

## Drafting Order

1. **Background** — the policy problem, current context, why action is needed now.
2. **Evidence base** — key findings organized by policy relevance.
3. **Policy options** — each option with pros, cons, cost, feasibility.
4. **Recommendations** — recommended option with justification, implementation steps, timeline, success metrics.
5. **Conclusion** — concise call to action.

## Core Principles

- **Stakeholder-aware framing**: write for decision-makers, not for academic peers. The audience needs actionable evidence, not theoretical nuance.
- **Evidence-to-recommendation chain**: every recommendation must trace back to specific evidence. If a recommendation lacks evidential grounding, flag it explicitly.
- **Feasibility matters as much as effectiveness**: an option that works in theory but is politically or financially infeasible should be noted, not hidden.

## Evidence Base Structure

Organize findings by policy relevance:

1. **Key finding 1** — with direct policy implication.
2. **Key finding 2** — with supporting evidence and caveats.
3. **Key finding 3** — with comparative or international context if relevant.

## Policy Options Analysis

For each option:

| Dimension | Assessment |
|-----------|-----------|
| Description | What the option entails |
| Pros | Evidence-supported benefits |
| Cons | Evidence-supported drawbacks |
| Cost | Financial and resource requirements |
| Feasibility | Political, institutional, temporal constraints |

## Recommendations Structure

1. **Recommended option** — with justification from evidence.
2. **Implementation steps** — sequenced, actionable.
3. **Timeline** — realistic milestones.
4. **Success metrics** — measurable indicators of policy impact.

## Common Pitfalls

- Evidence presented without policy implications (academic findings that don't translate to action).
- Recommendations without feasibility analysis.
- Ignoring political or institutional constraints.
- Executive summary that reads as an abstract rather than a decision tool.
- Options that are straw-men rather than genuine alternatives.
