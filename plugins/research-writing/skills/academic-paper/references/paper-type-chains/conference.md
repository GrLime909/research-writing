# Paper Type: Conference Paper

Corresponds to `paper_structure_patterns.md` Pattern 6.

## Argument Chain

`task -> gap -> proposed method/approach -> evaluation -> failure modes -> future work`

## Drafting Order (time-constrained)

Conference papers are shorter and faster to produce. Draft in contribution-first order:

1. **Core contribution** — what is new, written first to anchor everything else.
2. **Results/Evaluation** — the evidence that supports the contribution.
3. **Introduction** — frames the task and gap retrospectively around the contribution.
4. **Related Work** — positioned against the contribution.
5. **Methodology** — written for reproducibility within page limits.
6. **Conclusion** — concise summary + future work.
7. **Abstract** — last, distilled.

## Core Principles

- **Contribution-first**: conference reviewers scan for novelty quickly. Make the contribution unmistakable in the first two pages.
- **Compact over complete**: a focused paper with one strong contribution beats an overextended paper with three weak ones.
- **Fail fast on structure**: if the contribution cannot be stated in 2-3 sentences, the paper is not ready.

## Section Budget (for 8-page conference paper)

| Section | Approximate % |
|---------|--------------|
| Introduction | 15% |
| Related Work | 15% |
| Methodology | 25% |
| Results | 25% |
| Discussion | 12% |
| Conclusion | 8% |

## Conference-Specific Rules

- Respect page limits strictly. Do not try to squeeze content via font size or margin tricks.
- Figures and tables must be self-contained — a reviewer reading only the figures should understand the contribution.
- Related Work should be tight: 2-3 focused paragraphs, not a comprehensive survey.
- The Discussion section can be shorter than in journal papers — acknowledge limitations and move on.

## Common Pitfalls

- Overpromising in the introduction relative to what the page limit allows.
- Comprehensive Related Work that crowds out the contribution.
- Methodology written at journal length then brutally cut, losing reproducibility.
- Results without failure mode analysis — reviewers trust papers that report their own limits.
- Conclusion that merely repeats the abstract.
